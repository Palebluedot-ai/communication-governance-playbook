# Global Goal

## 这库到底在做什么

这个库要做的，不是单纯整理一套聊天模板，也不是先堆一堆脚本。

它要做的是一个 **AI 协作里的防漂移控制系统**：

- 让 AI 在长对话里不要偷偷换目标
- 让澄清问题不会自动变成切阶段
- 让开发过程始终记得当前主线，而不是被局部细节带走
- 让每一轮协作都能回看、检查、复现

一句话说：

> 把“对话不跑偏”从习惯，做成可执行、可检查、可回拉的控制系统。

## 这个系统最终要控制什么

重点控制 3 件事：

1. `Goal` 不能在对话里被悄悄改掉
2. `Next` 不能因为澄清、补充、灵感、追问就自动切换
3. 开发流程不能只顾局部细节，丢掉全局路线图

## 这个系统最后要嵌进哪里

它不是独立悬空存在的，要嵌进这条开发链路里：

`需求澄清 -> 任务拆解 -> MRD -> BMaD 开发`

也就是说，这个库要保证上面这条链路在推进时始终受同一主线约束。

## 非目标

当前明确不是下面这些：

- 不是先做跨平台全家桶接入
- 不是先做 Hermes / Discord / Cloud DROID 的深度集成
- 不是先堆很多 runtime 文件来制造“已经实现”的感觉
- 不是为了写文档而写文档

## 成功的样子

这个库第一阶段成功，不看文件数量，看 4 件事：

1. 当前 `Global Goal` 能被稳定复述，不会一问一答就变形
2. 当前阶段明确，澄清不会自动切阶段
3. 每一步开发都有对应文档，不再只存在聊天记录里
4. 后续真的开始做执行层时，是按已确认文档往下做，不是边聊边飘
